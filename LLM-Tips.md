# LLM 交互提示词（项目级上下文）

> 本项目特定的 LLM 提示词模板，用于在与 AI 协作时快速建立正确的上下文边界。
> 建议在与新会话的 LLM 交互时，将以下内容作为前置提示发送。

---

## 代码总结 / 分析场景

```text
请总结这个项目的代码，请务必忽略 @.gitignore 中的目录或文件，请务必忽略 .git 目录。这里面混杂大量二进制会巨量消耗 Token。
```

### 说明
- `.gitignore` 中忽略的目录/文件（如 `obj/`, `bin/`, `*.o`, `test_output/` 等）通常是构建产物或临时文件，对代码理解无价值。
- `.git/` 目录包含 Git 对象数据库，对于大型项目可能包含数 MB 甚至数十 MB 的二进制对象，会迅速耗尽上下文窗口。
- 本项目为 C 语言项目，核心源码位于 `src/` 和 `include/`，其余多为构建产物或设计文档。

---

## Git 操作场景

```text
请将本次更新内容纳入git管理域（git add），并将本次更新提交到git，并push到远端。请将本条提示词先更新到LLM-Tips.md中。
```

### 说明
- 在与 LLM 协作过程中产生的新文件或修改，应及时纳入版本控制。
- `git add` 前应确认 `.gitignore` 已正确配置，避免提交构建产物。
- 提交信息应简洁描述变更内容（如 `docs: add BDD spec and cursor fixme`）。
